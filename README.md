# LSL Blender Animation Pipeline

Procedural animation pipeline for Latvian Sign Language (LSL). Pose data (hand shapes, arm locations, wrist orientations) is stored as JSON files, assembled into per-sign animation clips in Blender via IK constraints, baked to bone transforms, and exported as a single GLB for the web viewer.

### Read the Handbook here: https://lucyferyx.github.io/LSL-handbook

---

## Project Structure

```
blender_project/
├── LSL_programmatical.blend        ← Blender scene with rigged character + IK empties
├── lsl_model.glb                   ← Exported output (auto-generated by bake)
├── signs.json                      ← Sign library: maps sign names to poses + movements
├── main_blender_script.py          ← Entry point — paste into Blender Scripting tab
│
├── poses/
│   ├── LSL_Start_Position.json     ← Full neutral pose (arm IK + fingers)
│   ├── LSL_Hand_Relaxed.json       ← Relaxed transition pose (finger waypoint)
│   ├── LSL_Hand_A.json             ← Fingerspelling / vocabulary hand shapes
│   ├── LSL_Hand_B.json             ← (one file per hand shape)
│   ├── LSL_Location_Neutral.json   ← Arm + hand IK target positions
│   ├── LSL_Location_LeftTilted.json
│   ├── LSL_Orientation_Upwards.json    ← Wrist rotation presets
│   ├── LSL_Orientation_Backwards.json  ← (one file per orientation)
│   └── ...
│
└── scripts/
    ├── config.py           ← Constants: armature name, empty object names, POSES_DIR
    ├── pose_io.py          ← load_pose / save_pose / apply_world_transform / helpers
    ├── animation_moves.py  ← Procedural movements: slide, checkmark, halfcircle, etc.
    ├── sequence.py         ← create_sequence (multi-sign viewport preview)
    ├── bake.py             ← bake_all_signs_to_glb (full IK bake → NLA → GLB export)
    └── utils_json_edit.py  ← Standalone script to batch-update signs.json durations
```

## Setup

1. Open `LSL_programmatical.blend` in Blender 5.0+.
2. Go to the **Scripting** workspace and open `main_blender_script.py`.
3. Ensure `signs.json`, the `poses/` folder, and the `scripts/` folder are all in the **same directory as the `.blend` file** — the script resolves all paths relative to the `.blend`.

---

## Run / Bake

Running `main_blender_script.py` calls `bake_all_signs_to_glb()`, which:

1. Loads `signs.json`
2. For each sign: keyframes the IK empties, bakes the IK-solved bone transforms frame-by-frame to an NLA track
3. Exports all NLA tracks to `lsl_model.glb`

**Useful commands to uncomment in `main_blender_script.py`:**

```python
# Bake everything:
bake_all_signs_to_glb()

# Bake just one sign (faster iteration):
bake_all_signs_to_glb(signs_filter=["a"])

# Preview a sentence in the viewport without baking:
create_sequence("a ā b c")

# Reset viewport animation:
reset_animation()
```

## Adding a New Sign

### 1. Capture the hand shape

Set up the right hand in the viewport (adjust `Empty_RightHand*` finger empties), then:

```python
save_pose("Hand_MyShape", include_arm=False)
```

This writes `poses/LSL_Hand_MyShape.json`.

### 2. Add the sign to `signs.json`

```json
"my_sign": {
    "right": {
        "shape": "Hand_MyShape",
        "location": "Location_Neutral",
        "orientation": "Orientation_Upwards",
        "move": null
    },
    "left": { "shape": "Start_Position" }
}
```

**Field reference:**

| Field | Required | Description |
|---|---|---|
| `shape` | yes | Hand pose filename suffix (`Hand_A` → `LSL_Hand_A.json`) |
| `location` | yes (active sides) | IK target positions (`Location_Neutral` for standard signing space) |
| `orientation` | yes | Wrist rotation preset |
| `move` | no | `null`, or `{"type": "slide", "direction": "left"}` |
| `duration` | no | Hold frames (default 10) |
| `head` | no | Head action (`"Head_Nod"`) |
| `left: "mirror_right"` | — | Mirror the right-hand pose to the left |

Inactive hand (stays at rest): `{ "shape": "Start_Position" }` — omit `location` and `orientation`.

### 3. Bake

```python
bake_all_signs_to_glb(signs_filter=["my_sign"])
```

## Adding a New Animation Movement

1. Add a function `animate_<name>(start_frame, duration, side="right")` in `scripts/animation_moves.py`.
2. Wire it up in `scripts/sequence.py` (`create_sequence`) and `scripts/bake.py` (`_animate_sign_for_baking`) — follow the existing `elif move_type == "..."` pattern in both places.
3. Set `is_movement = True` (and `protect_rotation = True` if the move changes wrist rotation during hold).
4. Use it in `signs.json`:
   ```json
   "move": { "type": "my_move_name", "direction": "left" }
   ```

## Capturing a New Location

A **location** defines where the arm and hand sit in world space (IK target positions). Most signs use `Location_Neutral`.

1. Position `Empty_RightArm` and `Empty_RightHand` in the viewport.
2. Run:
   ```python
   save_location_pose("MyLocation")
   ```
   This writes `poses/LSL_Location_MyLocation.json`.
3. Use `"location": "Location_MyLocation"` in `signs.json` for any sign at that position.

---

## Capturing a New Orientation

An **orientation** defines the wrist rotation: forearm twist (`Empty_RightForearm`) and hand rotation (`Empty_RightHand`).

1. Set the desired wrist rotation in the viewport (rotation only — location is handled by the Location file).
2. Run:
   ```python
   save_orientation_pose("MyOrientation")
   ```
   This writes `poses/LSL_Orientation_MyOrientation.json`.
3. Use `"orientation": "Orientation_MyOrientation"` in `signs.json`.

## How the Pose System Works

Each sign frame is assembled from three layers loaded in order:

| Layer | File type | Controls |
|---|---|---|
| Location | `LSL_Location_*.json` | Where the arm and hand sit (IK target XYZ) |
| Orientation | `LSL_Orientation_*.json` | How the wrist is rotated (forearm twist + hand rotation) |
| Shape | `LSL_Hand_*.json` | Finger positions |

Right side is always loaded before left so that `_RIGHT_HAND_WORLD_REF` is set before the left-hand finger mirroring math runs.
